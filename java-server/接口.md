# 暗盘榜

- **接口路径**: `/indicator/capital/v1/grey_rank`
- **请求方法**: GET
- **状态**: done

## 接口描述

暗盘资金榜单接口：根据一定规则，生成暗盘资金个股、板块榜单列表。

`data.stock_list` 支持分页返回，并包含索引字段。
全量 `stock_list` 在服务端按 `turnover` 降序后再分页返回。

## 请求参数

| 参数名 | 类型 | 必填 | 示例值 | 描述 |
|--------|------|:----:|--------|------|
| date | date | 否 | 2025-03-04 | 日期，格式如 2025-03-04，若不传默认最新一个交易日 |
| pageNum | integer | 否 | 1 | 页码（从 1 开始），默认 1 |
| pageSize | integer | 否 | 20 | 每页条数，默认 20 |

## 响应状态码

| code | value | 描述 |
|:----:|-------|------|
| 0 | success | 请求成功 |
| 401 | denied | 没有权限 |
| 500 | internal error | 服务器内部错误 |

## 响应数据结构

```json
{
  "status_code": 0,
  "status_msg": "请求成功",
  "data": {
    "total": 60,
    "stock_list": [...],
    "block_list": [...],
    "update_time": "2025-04-01 15:00"
  }
}
```

### data.stock_list - 暗盘榜-个股

| 字段 | 类型 | 描述 |
|------|------|------|
| stock_code | string | 股票代码 |
| stock_name | string | 股票名称 |
| market | string | 股票市场 |
| main_grey_capital | number | 主力暗盘资金 |
| main_listed_capital | number | 主力明盘资金 |
| main_capital | number | 主力净流入(等于主力暗盘+主力明盘) |
| retail_capital | number | 散户流入(等于主力净流入的负数) |
| turnover | number | 成交额 |
| index | integer | 当前记录在全量结果中的序号（从 1 开始） |
| ratio_pct | number | 占比(主力明盘资金/成交额 * 100，保留 2 位小数) |
| zf | number | 涨幅 |

### data.block_list - 暗盘榜-板块

| 字段 | 类型 | 描述 |
|------|------|------|
| block_code | string | 板块代码 |
| block_name | string | 板块名称 |
| market | string | 板块市场 |
| zf | number | 涨幅 |
| member_stock_cnt | integer | 总家数(成分股个数) |
| main_grey_capital_in_cnt | integer | 主力暗盘资金流入家数 |
| main_grey_capital_in | number | 主力暗盘资金流入 |
| main_grey_pct | number | 主力暗盘占成交额 |
| stocks | array | 涨幅最高的2个成分股 |

#### stocks 数组字段

| 字段 | 类型 | 描述 |
|------|------|------|
| stock_code | string | 股票代码 |
| stock_name | string | 股票名称 |
| market | string | 市场 |
| zf | number | 涨幅 |

### data.update_time

| 字段 | 类型 | 描述 |
|------|------|------|
| update_time | string | 更新时间 |

## 分页行为说明

- 分页参数：`pageNum >= 1`，`pageSize >= 1`。
- `total` 表示全量个股条数（分页前）。
- 当 `pageNum` 超过最大页时，返回空 `stock_list`，`total` 仍为真实总数。
